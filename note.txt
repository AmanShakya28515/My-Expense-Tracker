 What is os.environ?
os.environ is a Python dictionary that gives you access to environment variables.

‚úÖ Environment Variables:
They are secure key-value pairs used to store secret or project-specific information ‚Äî like:

üîê SECRET_KEY

üåç WEBSITE_HOSTNAME

üîë Database passwords

üîß API keys

Let's Explain Line by Line:
1Ô∏è‚É£ SECRET_KEY = os.environ['SECRET']
In Django, SECRET_KEY is required for cryptographic signing (security of cookies, sessions, CSRF, etc.).

Instead of writing it directly in your settings.py (like SECRET_KEY = 'abc123...'), you're pulling it from Azure's secure environment:

SECRET_KEY = os.environ['SECRET']
‚úÖ This keeps your key safe and hidden ‚Äî good for security and deployment.

2Ô∏è‚É£ ALLOWED_HOSTS = [os.environ['WEBSITE_HOSTNAME']]
ALLOWED_HOSTS defines which domain names are allowed to access your app.

‚úÖ On Azure, the app gets a domain like:

yourapp.azurewebsites.net
So Azure creates an environment variable called WEBSITE_HOSTNAME, and this line pulls it:

ALLOWED_HOSTS = ['yourapp.azurewebsites.net']
‚úÖ This prevents Django from throwing a "Bad Request (400)" error.

3Ô∏è‚É£ CSRF_TRUSTED_ORIGINS = ['https://' + os.environ['WEBSITE_HOSTNAME']]
This allows your domain to be trusted for CSRF protection (important when submitting forms or using login/logout).

‚úÖ For example:

CSRF_TRUSTED_ORIGINS = ['https://yourapp.azurewebsites.net']
üîí It tells Django: "This origin is trusted, don't block CSRF requests from here."

1Ô∏è‚É£ 'corsheaders.middleware.CorsMiddleware'
‚úÖ Allows your Django backend to accept requests from other domains (e.g., frontend on Vercel).
‚úÖ Use CORS_ALLOWED_ORIGINS to whitelist allowed origins.

2Ô∏è‚É£ 'whitenoise.middleware.WhiteNoiseMiddleware'
‚úÖ Serves static files (CSS, JS, images) directly in production.
‚úÖ No need for extra servers like NGINX.

3Ô∏è‚É£ 'django.middleware.security.SecurityMiddleware'
